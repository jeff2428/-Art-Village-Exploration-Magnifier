# 軟體 QA 檢視清單

這份清單用於本機或部署後檢視「藝素村探險放大鏡」功能、流暢性與視覺品質。每次改動 UI、相機、辨識、圖鑑或部署流程後，都應至少跑一次核心路徑。

## 啟動與首頁

- [ ] 本機啟動：`flet run -w -p 8550 --host 127.0.0.1 main.py`
- [ ] 瀏覽器開啟 `http://127.0.0.1:8550` 回應 200，首頁標題與開始按鈕可見。
- [ ] 點擊「開始探險」後，loading 訊息循環顯示，最後進入主畫面。
- [ ] 若相機元件或權限不可用，狀態列顯示友善提示，不出現空白畫面或未處理錯誤。

## 植物模式

- [ ] 預設進入植物模式，放大鏡、拍攝按鈕、縮放按鈕、部位選擇器與圖鑑區塊都可見。
- [ ] 部位選擇器可在自動、葉、花、果、樹皮間切換，選取狀態清楚。
- [ ] 縮放按鈕只更新鏡頭/把手/狀態，不造成整頁明顯跳動。
- [ ] 相機尚未就緒時點拍攝，會顯示「正在重新啟動」或「準備中」提示。
- [ ] Worker 未設定或辨識失敗時，顯示繁中友善錯誤，不暴露原始 API payload。

## 動物模式與圖鑑

- [ ] 切到動物模式後，相機 preview 暫停/隱藏，動物卡片清單顯示。
- [ ] 點擊動物卡片會打開介紹卡，並加入探險圖鑑。
- [ ] 圖鑑卡片 hover 有輕微放大與陰影變化，不卡頓、不重排。
- [ ] 長按圖鑑卡片會出現刪除確認，刪除後卡片消失並保存快取。
- [ ] 清除按鈕在空圖鑑時顯示提示，有資料時顯示確認 dialog 並可清空。

## 視覺與響應式

- [ ] 淺色與深色模式都能切換，文字對比清楚，主要動作色與資訊色容易辨識。
- [ ] 主要卡片圓角、陰影、間距一致；植物卡、動物卡、圖鑑卡看起來屬於同一套 UI。
- [ ] 寬度約 360px、430px、768px 檢視時，標題、按鈕、卡片文字沒有重疊或溢出。
- [ ] 首屏能看到核心操作，不需要先滾動才知道如何開始。

## 效能驗收

- [ ] 依 `docs/performance-baseline.md` 量測縮放、卡片動畫與圖鑑刷新。
- [ ] 桌面目標：卡片動畫平均至少 50 fps；縮放點擊到畫面更新接近單一影格。
- [ ] 行動裝置目標：卡片動畫平均至少 30 fps；模式切換無明顯卡住。
- [ ] 若效能不達標，優先檢查整頁 `page.update()`、過重陰影、過多 stagger update。
